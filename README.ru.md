# SpecFly

SpecFly — мультиязычная спецификационная система, где язык пользователя является первым языком взаимодействия с системой и агентом, а не переводом поверх английской модели.

## Манифест

### 1. User-first language

Спецификации создаются на языке, удобном пользователю.
Работа с системой должна быть прозрачной и предсказуемой.

### 2. Shared vocabulary

Проектный глоссарий — часть системы.
Все доменные термины фиксируются и используются последовательно.

### 3. Dual representation

Каждая спецификация существует в двух представлениях:

- Human view — для человека
- Agent view — для агента

Это две проекции одной спецификации, а не два независимых документа.

### 4. Change-first workflow

Новая фича, исправление или улучшение сначала живут как изменение.
После реализации и проверки изменение интегрируется в актуальную спецификацию.

### 5. Task derivation

Задачи должны выводиться из спецификации.
Если из спеки нельзя получить понятный план работ, значит она недостаточно качественная.

### 6. Validatable specs

Спецификация должна быть машинно проверяемой.
Структура, ссылки, обязательные секции, конфликты и задачи должны валидироваться.

### 7. Explicit decisions

Агент не должен молча принимать важные решения.
Архитектурные, UX, data и security-решения должны фиксироваться явно.

### 8. Executable contract

Спецификация — это исполняемый контракт.
Если для реализации или проверки нужны инструменты, они являются частью системы.

### 9. AI-optional

Система работает с AI.
Система работает без AI.
